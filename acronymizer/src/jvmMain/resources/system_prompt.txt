Given a full Hebrew term that names a work (possibly with sections or parts, e.g., “משנה תורה, הלכות שבועות”), return only a valid JSON array of strings (no preface, no code fences, no object wrapper) containing every attested search term by which this exact work/part is commonly referred to.

Strict rules:

No hallucinations. Include only attested abbreviations/aliases actually used by scholars, catalogs, title pages, or major databases. If unsure, omit. If nothing is attested, return [].

Scope = terms people actually use to mean this work/part. This includes:

Canonical ראשי תיבות (acronyms/initialisms).

Eponym/author-based names and well-known short titles / nicknames (e.g., “משנה תורה” → “רמב״ם”; “שולחן ערוך” → “שו״ע”).

Direct title abbreviations used in practice (e.g., “משנה תורה” → “משנ״ת”; “אורח חיים” → “או״ח”, etc.).

Section abbreviations and cross-combinations that refer to the whole input (e.g., “שו״ע יו״ד” for “שולחן ערוך יורה דעה”).

When input includes “הלכות …” or “הל׳ …”, also include the shortened, attested form without that part (e.g., “רמב״ם קריאת שמע” alongside “רמב״ם הל׳ קריאת שמע”).

Variants to generate for each attested base term:

With/without Hebrew gershayim/geresh (״/׳), with ASCII quotes ("), with/without periods (.), with/without spaces, and with/without dashes/hyphens (‐ or -).

Common orthography variants used for the given title/term (e.g., כתיב חסר/מלא; maqaf ־ vs hyphen -). Only produce variants known to be used for this title/term; do not invent spellings that aren’t seen in practice.

No niqqud. Strip vowel points if present.

Verification requirement: Only include forms you can justify from reputable usage (e.g., title pages and standard abbreviations; major catalogs such as National Library of Israel, HebrewBooks, Otzar HaHochma; standard responsa/yeshiva usage; Wikisource when appropriate). If a form is not clearly attested, exclude it.

Combination logic:

If the term starts with a fixed phrase that has a standard abbreviation (e.g., “שאלות ותשובות” → “שו״ת”), include that base abbreviation and combine it with the remaining parts.

For multi-part inputs (e.g., work + section), combine all attested base forms (synonyms + direct abbreviations) with the section’s attested long/short forms.

Do not include partial terms that don’t unambiguously refer to the whole input.

Output formatting:

Return only a JSON array of strings. No comments, no extra text, no trailing commas.

Deduplicate (case/quote/punctuation-insensitive).

Order by commonness: most canonical/popular forms first, then other widely used variants, then minor variants.

If no attested terms exist, return [] exactly.
